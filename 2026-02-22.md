## RESTful API 리팩토링

```
최종목표 : Restful한 API로 수정

- backend 프로젝트를 분석하고 현재 문제점 파악.
```

### HTTP 상태값에 대한 이해

#### HTTP 201

201은 리소스가 새로 생겼다는 의미.

[201 응답에 꼭 Location이 있어야 할까](https://chatgpt.com/share/699a8ed2-bd1c-8003-bd09-f09a355c384d)

```java
URI location = ServletUriComponentsBuilder.fromCurrentRequest()  
		.path("/{id}")  
		.buildAndExpand(response.id())  
		.toUri();  

return ResponseEntity.created(location).body(response);  
```

`ServletUriComponentsBuilder`를 활용하여 Location 헤더를 넣는다.

> 단순한 경로 제공, HTTP 메서드는 제공하지 않음.

추가로 SwaggerUI에 헤더를 명시하기 위해서는 `@ApiResponse`에 추가로 작성이 필요하다.

```java
@ApiResponses(value = {  
        @ApiResponse(  
                responseCode = "201",  
                description = "생성 성공",  
                headers = @Header(
	                name = "Location", 
	                description = "생성된 운동 리소스 URI"
				)  
        )
})
```

#### HTTP 204 

데이터 삭제 후 삭제 작업이 완료된다면 클라이언트에게 보내줄 정보가 없긴 함.  
그렇기에 204를 반환하도록 수정.
