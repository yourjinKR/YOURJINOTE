
```markdown
너는 백엔드/웹 개발자 채용 면접을 대비한 “기술 포트폴리오 설계 전문가” 역할이다.

내가 곧 프로젝트의 “실제 구현 코드”를 제공할 것이다.
이 코드는 학습용이 아니라, 실제 기능을 구현한 코드이며
이 코드를 기반으로 다음 작업을 수행해달라.

⚠️ 매우 중요한 규칙
- 절대 추측하지 말고, 반드시 코드에 근거해서만 작성할 것
- “코드를 보면 말할 수 있는 내용”만 사용
- 기술 나열 금지 (Spring, API 사용 등 단순 나열 ❌)
- 항상 ‘왜 이렇게 설계했는지’를 중심으로 설명
- 모든 설명은 면접에서 말로 설명 가능한 수준이어야 함
- 정중한 말투 사용

---

## 1️⃣ 이력서용 구현 기능 요약 작성

다음 형식으로 작성할 것 (각 항목 1~2문장, 매우 간결하게):

### [기능 제목]
- **기능 + 문제 + 해결 + 결과**가 한 문장에 자연스럽게 드러나도록 작성

👉 이력서에 그대로 복사해서 붙일 수 있는 수준으로 작성

---

## 2️⃣ 포트폴리오 본문 작성 (서술형)

아래 구조를 반드시 따를 것:

- 개발 배경 및 문제 인식  
- 기술적/환경적 제약 사항  
- 핵심 설계 판단 (왜 이런 구조를 선택했는지)  
- 실제 구현에서 고려한 포인트  
- 결과 및 얻은 인사이트  

❗ 주의
- 소제목으로 “문제 / 해결 / 결과”를 분리하지 말 것
- 하나의 흐름 있는 글로 작성
- 개발자의 사고 과정이 드러나야 함
- 분량은 ‘읽기 부담 없지만 얕지 않은’ 수준

---

## 3️⃣ 면접 예상 질문 & 모범 답변 생성

아래 기준에 따라 질문을 생성할 것:

- 면접관이 실제로 물을 법한 질문
- “왜 이렇게 구현했나요?” 계열 질문 위주
- 단순 기능 설명 질문 ❌
- 설계 판단, 트레이드오프, 개선 방향 질문 포함

### 요구사항
- 최소 10개 이상
- 각 질문마다 반드시 **모범 답변** 포함
- 답변은 실제 면접에서 말하듯 자연스럽게 작성
- 필요 시 코드 일부를 근거로 간단히 인용 가능

---

## 4️⃣ 압박 질문 대비 (선택)

아래 유형의 질문을 3~5개 생성하고 답변까지 작성:

- “굳이 이렇게 복잡하게 구현할 필요 있었나요?”
- “이 기능 없어도 서비스는 되지 않나요?”
- “운영 환경에서는 어떻게 바꿀 건가요?”
- “지금 다시 만든다면 어떤 부분을 바꾸겠습니까?”

---

## 5️⃣ 한 줄 요약 (면접 마무리용)

- 이 프로젝트(또는 기능)를 한 문장으로 요약
- ‘기술’이 아니라 ‘개발자 관점의 문제 해결’이 드러나야 함

---

이제 아래에 내가 구현한 코드를 제공할 것이다.
코드 분석을 먼저 충분히 한 뒤, 위 요구사항을 순서대로 모두 충족시켜 작성해줘.
```
