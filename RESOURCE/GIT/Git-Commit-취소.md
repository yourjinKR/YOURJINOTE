

# revert

```bash
# 특정 커밋 되돌리기 (새 커밋 생성)
git revert <commit-id>
```

# reset

```bash
# 바로 이전 커밋 취소 (스테이징 유지)
git reset --soft HEAD~1

# 바로 이전 커밋 취소 + 스테이징도 해제
git reset --mixed HEAD~1

# 바로 이전 커밋 취소 + 변경사항도 삭제
git reset --hard HEAD~1
```

# revert vs undo

| 구분      | `git revert`                       | `git reset` (혹은 `undo commit`)     |
| ------- | ---------------------------------- | ---------------------------------- |
| 목적      | **되돌리는 새 커밋을 생성**                  | **이전 커밋 상태로 되돌리고 기록 수정**           |
| 작동 방식   | 기존 커밋의 변경 사항을 **반대로 적용한 새 커밋을 만듦** | HEAD를 이전 커밋으로 이동시켜 **히스토리 자체를 변경** |
| 히스토리    | 안전하게 유지됨 (공유 브랜치에 안전)              | 히스토리가 바뀜 (공유 브랜치에 위험)              |
| 사용 예시   | 잘못된 코드가 머지된 이후 → 되돌리고 싶을 때         | 최근 커밋(아직 push 전)을 취소하고 싶을 때        |
| 협업 환경   | ✅ 안전 (공유 브랜치 가능)                   | ⚠️ 위험 (로컬에서만 안전)                   |
| 결과      | "Revert '커밋메시지...’" 형태의 새 커밋 생성    | 과거 커밋 시점으로 브랜치가 이동, 중간 커밋 사라짐      |
| 되돌리는 대상 | 지정한 특정 커밋                          | HEAD 기준으로 되돌림(하나 혹은 여러 개)          |
