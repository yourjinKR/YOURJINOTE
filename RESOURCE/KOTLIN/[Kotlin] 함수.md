# 함수형 프로그래밍

## 기본적 사상
### 참조 투명성
외부의 값에 영향받지 않으며 동일한 입력에 항상 동일한 결과를 반환하는 성질입니다.
### 불변성
변수의 값을 한 번 설정한 후에는 변경할 수 없다는 개념입니다.
### 순수 함수
함수의 결과가 오직 입력에만 의존하고, 함수 호출이 부작용을 일으키지 않는 함수를 의미합니다.
```java
int add(int a, int b) {
	return a + b
}
```

다른 함수를 매개변수로 받거나 반환값으로 내놓는 함수
