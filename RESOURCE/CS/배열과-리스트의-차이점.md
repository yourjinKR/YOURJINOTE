
# 리스트와 배열의 차이점

[배열](Array-배열.md)은 동일한 데이터 타입의 요소들을 연속적인 메모리 공간에 저장하는 데이터 구조이다.  
배열의 각 요소는 인덱스로 식별되며, 이 인덱스는 요소의 위치를 나타낸다.  
배열의 크기는 초기화 과정에서 메모리 크기가 정해지기에 정적인 특성을 지닌다.

리스트는 배열과 달리 다양한 데이터 타입을 처리할 수 있는 유연성 있는 데이터 구조이다.  
배열과 달리 리스트는 동적이며 이러한 특성으로 인해 리스트는 요소 추가 또는 삭제와 같은 작업에 유용하다.

| 측면      | 배열                                      | 리스트                                      |
| ------- | --------------------------------------- | ---------------------------------------- |
| 크기      | 배열은 생성 시 크기가 고정됩니다.                     | 리스트는 동적이며 실행 중에 크기가 변경될 수 있습니다.          |
| 데이터 유형  | 배열의 모든 요소는 동일한 데이터 유형이어야 합니다.           | 리스트는 서로 다른 데이터 유형의 요소를 포함할 수 있습니다.       |
| 메모리 할당  | 배열 전체에 필요한 메모리는 초기화 과정에서 한 번에 할당됩니다.    | 리스트는 요소가 추가되거나 제거될 때 동적으로 메모리를 할당합니다.    |
| 접근 시간   | 배열은 인덱싱을 통해 요소에 상수 시간으로 접근할 수 있도록 해줍니다. | 목록은 동적 크기 조정으로 인해 접근 시간이 약간씩 달라질 수 있습니다. |
| 유연성     | 배열은 크기를 쉽게 변경할 수 없기 때문에 유연성이 떨어집니다.     | 리스트는 더 유연하여 요소를 쉽게 추가하거나 제거할 수 있습니다.     |
| 메모리 효율성 | 필요 이상으로 크기가 커지면 메모리 낭비로 이어질 수 있습니다.     | 동적 할당 덕분에 메모리 효율성이 더 높습니다.               |
| 일반적인 구현 | C/C++와 같은 언어에서 흔히 볼 수 있습니다.             | 파이썬이나 자바 같은 언어에서 흔히 볼 수 있는 현상입니다.        |
