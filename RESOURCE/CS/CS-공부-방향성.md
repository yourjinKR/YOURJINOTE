# 자료구조 가이드
### 웹 개발자를 위한 자료구조/알고리즘 공부 순서 가이드 🚀

1. **기본 개념 이해 및 복습 (자료구조)**
    
    - **배열 (Array)**: 가장 기본적인 자료구조로, 동적 배열(ArrayList 등)의 동작 방식과 시간 복잡도를 이해하는 것이 중요합니다.
    - **연결 리스트 (Linked List)**: 삽입/삭제 연산이 빈번한 경우에 사용되므로, 단일/이중 연결 리스트의 개념과 시간 복잡도를 파악해야 합니다.
    - **스택 (Stack)**: 함수 호출 스택, 웹 브라우저의 뒤로 가기/앞으로 가기 기능 등에서 활용됩니다. LIFO(Last-In, First-Out) 개념을 이해하세요.
    - **큐 (Queue)**: 비동기 작업 처리, 요청 큐 등에서 사용됩니다. FIFO(First-In, First-Out) 개념을 익히세요.
    - **해시 테이블 (Hash Table) / 해시 맵 (Hash Map)**: **매우 중요합니다!** 웹 개발에서 데이터를 빠르게 검색하고 저장하는 데 핵심적인 역할을 합니다. 키-값 쌍으로 데이터를 저장하며, 충돌 해결 방식(Chaining, Open Addressing)에 대한 이해가 필요합니다. (예: Java의 HashMap, Python의 Dictionary)
    - **트리 (Tree)**:
        - **이진 트리 (Binary Tree)**: 기본 개념을 익힙니다.
        - **이진 탐색 트리 (Binary Search Tree, BST)**: 검색, 삽입, 삭제 연산의 시간 복잡도를 이해합니다.
        - **균형 이진 탐색 트리 (Balanced BST)**: AVL 트리, 레드-블랙 트리 등은 내부적으로 많이 사용되지만, **개념적으로 이해하는 정도**로 충분합니다. (예: Java의 TreeMap, C++의 std::map)
        - **트라이 (Trie)**: 문자열 검색, 자동 완성 기능 등에서 활용될 수 있습니다.
2. **시간 복잡도 및 공간 복잡도 이해 (Big-O Notation)**
    
    - 어떤 알고리즘이나 자료구조 연산이 얼마나 효율적인지를 나타내는 **Big-O 표기법**을 반드시 이해해야 합니다. O(1), O(log n), O(n), O(n log n), O(n^2) 등을 구분할 수 있어야 합니다.
    - 각 자료구조의 주요 연산(삽입, 삭제, 검색)에 대한 시간 복잡도와 공간 복잡도를 파악하는 것이 중요합니다.
3. **핵심 알고리즘 (웹 개발 연관성 높은 것 위주)**
    
    - **정렬 알고리즘 (Sorting Algorithms)**:
        - **버블 정렬, 선택 정렬, 삽입 정렬**: 기본 개념과 O(n^2)의 시간 복잡도를 이해합니다.
        - **퀵 정렬, 병합 정렬**: O(n log n)의 효율적인 정렬 알고리즘으로, 내부적으로 많이 사용되므로 원리를 이해하는 것이 좋습니다.
    - **탐색 알고리즘 (Searching Algorithms)**:
        - **선형 탐색 (Linear Search)**: O(n)
        - **이진 탐색 (Binary Search)**: **매우 중요합니다!** 정렬된 데이터에서 O(log n)의 효율로 검색할 때 사용됩니다.
    - **그래프 알고리즘 (Graph Algorithms)**:
        - 그래프의 기본 개념 (정점, 간선, 인접 행렬, 인접 리스트)을 이해합니다.
        - **깊이 우선 탐색 (DFS)**, **너비 우선 탐색 (BFS)**: 웹 크롤링, 경로 찾기, 소셜 네트워크 분석 등 다양한 분야에서 활용될 수 있습니다. **개념과 구현 방법을 익히는 것**이 좋습니다.
        - 최단 경로 알고리즘 (다익스트라, 플로이드-워셜) 등은 백엔드 개발에서 직접적으로 사용될 일이 많지는 않지만, 알고리즘적 사고를 기르는 데 도움이 됩니다.
4. **실전 문제 풀이 (코딩 테스트 준비)**
    
    - 앞서 배운 자료구조와 알고리즘을 활용하여 실제 문제를 풀어봅니다.
    - **LeetCode**, **HackerRank**, **Programmers** 등의 플랫폼을 활용하여 웹 개발 관련 문제 유형(예: API 요청 처리, 데이터 파싱, 캐싱 전략 등)에 적용할 수 있는 문제들을 풀어보면 좋습니다.
    - 특히 해시 테이블, 이진 탐색, DFS/BFS 관련 문제들을 집중적으로 풀어보세요.

### 왜 이런 순서인가요?

- **기초부터 탄탄히**: 자료구조는 데이터를 어떻게 효율적으로 저장하고 관리할지에 대한 '그릇' 역할을 합니다. 웹 서비스에서 데이터를 다루는 백엔드 개발자에게는 이 '그릇'을 잘 이해하는 것이 가장 기본입니다.
- **효율성 측정**: Big-O 표기법은 어떤 '그릇'이나 '도구'를 선택하고 어떻게 사용하는 것이 효율적인지를 판단하는 기준이 됩니다.
- **핵심 도구 활용**: 정렬과 탐색 알고리즘은 데이터를 효율적으로 처리하는 핵심적인 '도구'들입니다. 특히 이진 탐색은 정렬된 데이터에서 검색 성능을 극적으로 향상시키므로 중요합니다.
- **응용력 강화**: 그래프 알고리즘은 좀 더 복잡한 문제 해결 능력을 길러주며, 다양한 서비스 로직에 응용될 수 있습니다.
