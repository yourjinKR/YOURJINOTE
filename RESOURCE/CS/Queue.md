# Queue

큐는 **FIFO(First In First Out)** 원칙을 따르는 선형 데이터 구조이다.  
데이터를 일시적으로 쌓아두기 위한 목적으로 주로 사용한다.

> 모든 큐가  FIFO 원칙을 따르지는 않음

## 특징

- 선입선출 - 가장 먼저 들어온 요소가 가장 먼저 처리
- 양쪽 끝의 역할이 다르다
	- front : 요소를 꺼내는 위치
	- rear : 요소를 넣는 위치
- 중간 삽입, 삭제 불가

## 종류

Queue는 대표적으로 4가지 유형으로 나뉜다.

- Simple Queue
- Circular Queue
- Priority Queue
- Double Ended Queue

단순 큐는 큐 ADT의 가장 간단한 형태입니다. 데이터를 후단에서 삽입하고 전단에서 제거합니다. 또한 **단순 큐는 선입선출(FIFO) 원칙을 엄격하게 준수합니다.**

원형 큐는 큐의 첫 번째 요소가 마지막 요소를 가리키는 원형 구조를 가지고 있어, **단순 큐에 비해 메모리 활용 효율이 뛰어납니다.**

[우선순위 큐](https://www.baeldung.com/cs/priority-queue)라는 특수한 큐는 큐에 있는 각 요소에 우선순위를 저장합니다. **우선순위 큐는 각 요소에 할당된 우선순위에 따라 삭제 및 삽입 작업을 처리합니다.**

**양방향 큐는 FIFO(선입선출) 규칙을 따르지 않습니다.** 앞쪽과 뒤쪽 모두에서 요소를 삽입하고 제거할 수 있습니다.

## 주요 연산

- `enqueue()` : 큐의 끝에 요소를 삽입합니다.
- `dequeue()` : 큐가 비어 있지 않으면 큐의 첫 번째 요소를 제거하고 반환합니다.
- `peek()` : 큐가 비어 있지 않으면 큐에서 요소를 제거하지 않고 반환합니다.
- `size()` : 큐에 있는 요소의 개수를 반환합니다.
- `isEmpty()` : 큐가 비어 있으면 true를 반환하고, 그렇지 않으면 false를 반환합니다.

## 구현

Queue 또한 배열과 연결 리스트를 기반으로 구현이 가능하다.

### 배열 기반 큐

일정 크기의 배열을 기반으로 구현한다.  

- 캐시 친화적이며 단순한 구조
- 크기 제한이 존재
- 동적 확장 시 배열 복사가 필요
- 원형 큐 구현을 해야 공간 낭비가 생기지 않음

#### 원형 큐

```css
[1][2][3][4][5]
front        rear

/* 삽입 삭제 진행 */

[ ][ ][ ][4][5]
front        rear      
```

배열 기반 큐에서 데이터가 삽입/삭제가 반복됐을때 아래와 같이 메모리 공간을 재사용하지 못한다.

이를 해결하기 위해 rear나 front가 배열 끝에 도달하면 다시 0번 인덱스로 돌아가는 구조로 개선한다.  
즉, 배열을 원처럼 취급함.

```css
[ ][ ][ ][ ][ ]
 ↑           ↑
front       rear
```

- enqueue 하면 rear가 오른쪽으로 이동
- 끝에 가면 앞(0)으로 점프
- dequeue 하면 front가 오른쪽으로 이동
- 끝에 가면 0으로 점프

### 연결 리스트 기반 큐

노드들을 체인처럼 연결하는 방식으로 구현한다.  
큐의 front를 head, rear를 tail로 관리한다.

```css
front → [Node1] → [Node2] → [Node3] ← rear
```

- 크기 제한이 없음
- 각 노드가 참조를 저장하기에 메모리 오버헤드 발생
- 노드가 메모리 여기저기에 흩어져 있어 캐시 측면에서는 비효율성 존재
- 동적 메모리 할당 자체에 대한 비용 존재

#### enqueue

- 새 노드를 만들고
- tail 뒤에 연결하고
- tail = 새 노드 로 업데이트

#### dequeue

- head를 제거
- head = head.next로 업데이트
