# 자바의 특징

자바의 특징을 파악하고 자바라는 언어를 이해하자.

## 장점
### 1. 간단하다
자바는 배우기 싶고 코딩하기 간결하도록 설계되었다.  
대표적으로 C와 C++의 포인터와 같은 복잡한 특징들을 전부 제거하여 코드를 더 쉽게 수행할 수 있게 만든다.
### 2. 객체지향 언어다
자바의 주요 장점은 [객체 지향 언어 ](OOP.md)라는 것이다.  
모든 것을 [객체](객체-Objcect.md)로 다루며, 이를 통해 코드의 재사용성을 높이고 유지보수를 용이하게 한다.
### 3. 안전하다
포인터를 사용하지 않아 메모리에 직접 접근할 수 없으며, [클래스 로더](Java-클래스-로더-Class-Loader.md)와 [바이트 코드 검증기](바이트-코드-검증기.md) 등을 통해 바이러스나 악성 코드로부터 비교적 안전합니다.

C와 C++과 같은 언어는 메모리 위치에 접근할 수 있는 포인트를 사용하는데 이는 보안 위험을 야기한다.  
자바는 캡슐화, 추상화, 상속과 같은 객체 지향 프로그래밍을 사용하여 보안을 강화하고 사용자의 무단접근을 방지한다.
> **메모리 보안에 대해**
> 메모리에 접근이 가능하다는 것은 해당 메모리에 있는 값을 직접 변경할 수 있다는 뜻이다.
> 대표적인 예시로는 **버퍼 오버플로**가 있다.

### 4. 싸고 경제적이다
자바는 [빌드](../JAVA/Java-빌드.md)가 간단하여 개발 및 유지 관리가 쉽고 저렴하다.  
특정 시스템 하드웨어 인프라에 의존하지 않아 경제적입니다.
[자세히](../JAVA/Java-자바가-경제적인-이유.md)

### 5. 플랫폼에 독립적이다
Java로 작성된 코드는 [JVM](../JAVA/JVM.md)이 설치된 어떤 운영체제에서도 실행된다.
> **WORA (Write One Run Anywhere)**

### 6. 고수준의 프로그래밍 언어이다
자바 프로그램은 자연어와 같은 고급 언어이다. 영어의 문법과 유사하여 기억하기 쉽다.

### 7. 이식성
자바는 [이식성](소프트웨어-이식성-software-portability.md)이 매우 뛰어난 언어이다.  
플랫폼에 독립적일 뿐만 아니라 실행에 특별한 하드웨어가 필요하지 않기 때문이다.

### 8. 가비지 컬렉션을 제공한다
자바에서는 C와 C++과는 달리 JVM 내부의 [가비지 컬렉션](Java-가비지-컬렉션-Garbage-Collection.md)이 자동으로 메모리를 관리한다.  
어떠한 클래스도 참조하지 않고 역참조가 필요한 객체가 있을 때마다 JVM이 자동으로 프로그램에서 해당 객체를 제거하므로 추가 코드를 작성할 필요가 없다.

### 9. 자바는 멀티스레드를 지원한다
Java는 멀티스레딩을 지원하는 프로그래밍 언어이다.  
한 번에 여러 스레드를 독립적으로 실행할 수 있다.  
스레드는 애플리케이션의 효율성과 성능을 높이기 위해 공통 메모리를 공유한다.

### 10. 자바는 안정적이다
자바는 정기적인 업데이트를 통해 버그를 제거하기에 안정적인 프로그래밍 중 하나이다.

### 11. 자바는 분상형 언어이다
자바는 여러 컴퓨터 간에 데이터와 프로그래밍을 공유하는 매커니즘을 갖추고 있다.  
이는 시스템의 성능과 효율성을 향상시킨다.  
자바는 RMI를 지원하여 분산 처리를 가능하게 한다. 또한, 소켓 프로그래밍과 COBRA 기술을 지원한다.

### 12. 효율적인 메모리 할당 전략을 제공한다
자바는 메모리를 주로 힙과 스택으로 영역을 나눈다.  
JVM이 필요에 따라 두 영역 중 하나에 메모리를 할당하며 이는 효율적인 메모리 관리에 도움이 된다.

> Stack

### 13. 방대한 커뮤니티
자바는 오래된 언어이며, 전 세계 어떤 프로그래밍 언어보다 큰 규모의 개발자 커뮤니티가 있다.

## 단점
### 1. 느리다
C와 C++같은 네이티브 프로그래밍 언어보다 메모리를 더 많이 사용한다.  
또한 코드를 기계어로 변환하는 인터프리터의 추가 작업으로 인해 느리다.
### 2. GUI 부족
자바는 GUI가 부족합니다.
> **GUI (Graphic User Interface)**
> 컴퓨터나 다른 디지털 기기와 사용자가 시각적인 요소를 통해 상호작용할 수 있도록 하는 인터페이스
### 3. 백업 기능 부재
자바는 사용자 데이터를 백업하는 기능이 전혀 없다.  
데이터 저장에 중점을 두고 있지만, 백업 기능을 통해 데이터를 보호하지는 않는다.
> 그럼 이를 어떻게 해결하는가?
### 4. 더 많은 메모리 공간 필요
자바는 많은 메모리를 사용한다. Garbage Collector를 사용하지만 이는 성능에  영향을 미친다.
### 5. 장황하고 복잡한 코드
자바는 매우 장황하고 복잡한 구문을 갖는다.
