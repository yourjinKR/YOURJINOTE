# Call By Value

Call by Value는 함수에 값을 전달할 때 값을 복사하여 전달하는 방식

- 원본 데이터를 보호할 수 있다
- 데이터의 크기에 따라 성능 저하의 원인이 될 수 있다

# Call By Reference

함수에 값을 전달할 때 메모리 주소를 전달하는 방식

- 큰 데이터를 효율적으로 관리
- 원본 데이터가 변경될 위험이 존재

# 자바에서

자바에서는 원시 타입을 제외한 모든 데이터들은 객체로 관리되며 대부분의 함수(메서드)가 Call By Reference 방식으로 작동된다.  

# 사용법 비교

어떤 방식으로 데이터를 함수에 전달할지는 다음과 같은 요소를 고려하자.

- 데이터의 크기
- 안정성
- 성능

> 예를 들어 원본 데이터를 변경하지 않고 단순히 읽기만 할 경우에는 Call By Value를 사용하고,  
> 원본 데이터를 변경해야 할 경우에는 Call By Reference 방식을 선택하는 것이 좋다.
