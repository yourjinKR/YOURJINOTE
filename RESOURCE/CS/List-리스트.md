
# List

리스트는 동일한 데이터 타입의 요소들이 순서대로 배열된 모음이다.

## 리스트의 종류

리스트에는 다음과 같이 세 가지 유형이 존재

- Ordered List
- Unordered List
- Indexed List

## 배열과의 관계

[배열](Array-배열.md)을 사용하여 리스트를 쉽게 구현 할 수 있다.  
그러나 프로그래밍 언어에서 배열은 고정된 크기를 지정해야 하며, 지정할 수 있는 최대 크기에 대한 명확한 제한도 없다.  
따라서 배열로 리스트를 구현할 경우에는 `MemeoryOverFlow`가 발생할 수 있다.  
반대로, 저장할 데이터 양이 적을 경우 사용 가능한 메모리를 효율적으로 활용하기 어렵다.  

이를 해결하기 위해 클래스(객체)를 사용하여 리스트를 구현한다.  
클래스(객체)는 동적이므로 런타임에 데이터를 추가할 수 있는 유연성을 제공한다.  

## 주요 연산

- `get()` : 리스트에서 지정된 위치의 요소를 반환합니다.
- `insert()` : 리스트의 임의의 위치에 요소를 삽입합니다.
- `remove()` : 비어 있지 않은 리스트에서 어떤 요소든 처음 나타나는 위치를 제거합니다.
- `removeAt()` : 비어 있지 않은 리스트에서 지정된 위치의 요소를 제거합니다.
- `replace()` : 임의의 위치에 있는 요소를 다른 요소로 바꿉니다.
- `size()` : 리스트의 요소 개수를 반환합니다.
- `isEmpty()` : 리스트가 비어 있으면 true를 반환하고, 그렇지 않으면 false를 반환합니다.
- `isFull()` : 리스트가 가득 찼으면 true를, 그렇지 않으면 false를 반환합니다. 고정 크기 구현(예: 배열 기반 리스트)에서만 사용 가능합니다.

## 구현

리스트를 구현하는데 사용할 수 있는 대표적인 데이터 구조에는 [배열](Array-배열.md)과 연결리스트가 있다.

### 배열

- 고정된 크기의 배열 선언
- 배열에 데이터가 가득 찰 경우에는 더 큰 크기의 새로운 배열을 만든 후 기존 값들을 복사하여 옮긴다.
- 즉, 삽입/삭제가 자루 발생한다면 비효율 발생

### 연결 리스트

- 각 요소를 노드로 나누며 각 노드는 데이터와 다음 노드의 주소를 저장
- 삽입/삭제에 유리한 구조
