
# 추상 자료형

데이터 구조에 대한 연산 및 동작 집합을 정의하는 **개념적 모델**이다.  
어떻게 구현되는지 또는 데이터가 메모리에 어떻게 구성되는지는 명시하지 않는다.  

## 특징

ADT는 데이터와 해당 데이터에 대한 연산을 하나의 단위로 캡슐화 하는 방법이다.  

- 추상화
- 개념화
- 견고함
- 캡슐화
- 데이터 추상화
- 데이터 구조 독립성
- 정보 은닉
- 모듈성

## Java에서 ADT

Java에서 ADT를 사용하는 주요 이유는 다음과 같다.

- 캡슐화 : 복잡한 구현 세부 사항을 깔끔한 인터페이스 뒤에 은닉
- 재사용성 : 외부 사용 방식을 변경하지 않고도 다양한 내부 구현
- 모듈성 : 로직을 분리하여 유지 관리 및 업데이트를 간소화
- 보안 : 직접적인 접근을 차단하고 버그 및 의도치 않은 변경을 최소화 및 데이터 보호
