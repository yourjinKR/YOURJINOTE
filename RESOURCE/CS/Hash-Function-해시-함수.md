# Hash Function

임의의 크기를 자니는 데이터를 고정된 길이의 데이터로 변환하는 함수이다.  
데이터의 **무결성** 검증, 비밀번호 저장, 데이터 검색 최적화 등 다양한 곳에서 사용된다.

해시 함수는 **일방향성**을 가지기 때문에 무결성을 보장한다.

> 해시 값으로 원본 데이터를 복구하는 것이 불가능

## Hash Collision

해시 함수를 통해 나온 해시값이 중복될때 이를 **해시 충돌**이라고 한다.  
이러한 해시 충돌을 방지하기 위해 충돌 확률이 낮은 해시 함수를 선택하고 추가적인 검증 절차를 마련하는 것이 중요하다.

> MD5는 취약점이 발견되어 보다 안전한 SHA-256 방식을 사용함

대표적인 해결 방법으로는 2가지가 있다.

- open addressing
- seperate chaining
### seperate chaining

겹쳤을때 각각의 버킷을 LinkedList로 관리함

### open addressing

비어있는 공간을 활용함
