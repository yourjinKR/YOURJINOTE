# 웹 서버와 애플리케이션 서버의 차이점

[웹 서버](Web-Server.md)는 **정적인 컨텐츠**를 클라이언트에게 전달하는 역할을 담당한다.

> 정적인 컨텐츠로는 주로 HTML 페이지, 파일, 이미지, 동영상이 있다.

반면, [WAS](WAS-Web-Application-Server.md)는 **동적인 컨텐츠**를 생성하여 제공하는 역할을 수행한다.

> 애플리케이션 서버 또한 웹 컨텐츠를 제공할 수 있지만, 주요 역할은 사용자 클라이언트와 비지니스 로직간의 상호 작용을 활성화하여 트랜잭션 결과, 의사 결정 지원 또는 실시간 분석을 지원하는 것이다.

|               | **웹 서버**                                         | **애플리케이션 서버**                                           |
| ------------- | ------------------------------------------------ | ------------------------------------------------------- |
| **다루는 태스크**   | 웹 서버는 간단한 요청에 대한 응답을 제공합니다.                      | 애플리케이션 서버는 데이터베이스, 서비스 및 엔터프라이즈 시스템의 더 복잡한 콘텐츠를 제공합니다.  |
| **사용되는 프로토콜** | 웹 서버는 주로 HTTP를 사용합니다. FTP와 SMTP도 지원합니다.          | 애플리케이션 서버는 많은 프로토콜을 지원합니다.                              |
| **콘텐츠 유형**    | 웹 서버는 HTML 페이지, 이미지, 비디오 및 파일과 같은 정적 콘텐츠를 제공합니다. | 애플리케이션 서버는 실시간 업데이트, 개인화된 정보 및 고객 지원과 같은 동적 콘텐츠를 제공합니다. |
| **멀티스레딩**     | 일반적으로 멀티스레딩을 사용하지 않습니다.                          | 멀티스레딩을 사용하여 요청을 동시에 처리합니다.                              |

> 엄밀히 말해 웹 서버는 애플리케이션 서버의 일반적인 하위 집합입니다.

<br>

# 웹 서버와 WAS를 분리하여 사용하는 이유

WAS는 복잡한 비지니스 요구사항을 만족하는 동적인 컨텐츠를 처리하기 위해서 나온 것이며 WAS가 동적인 컨텐츠와 일반적인 웹 컨텐츠(정적 컨텐츠)를 제공할 수 있다면, 왜 WAS만 사용하는 것이 아닌 별개로 분리하여 사용하는가?

### 그 이유는 보안

WAS는 데이터베이스와 직접적으로 통신하며 중요한 비지니스 로직을 처리하기 때문에, 보안상의 이슈로 외부에 직접 노출하지 않는 것이 바람직하다. 웹 서버를 앞단에 배치하고 WAS로의 접근을 제어함으로써 보안성을 강화 할 수 있다.
