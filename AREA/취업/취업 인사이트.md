# 필터링한 나의 생각
많은 인사이트가있지만 참고하되 결국 **나만의 이야기**를 작성하는 것이 중요하다고 생각한다.

## 포트폴리오
- 전체적으로 이모지 삭제
- 열정, 패기와 같은 추상적인 메세지 삭제
- 자기소개 절대 길게 작성하지 않는다
## 프로젝트
- 하나만 더 하자, 제대로 된 녀석으로
- "문제 + 해결 + 결과 + 기능" 에 초첨하여 간단히 서술
## 블로그
- 퀄리티 있는 포스팅을 만들자
	- 1달에 하나씩 하나의 주제를 포스팅
	- 지금 일기 형식으로 작성하는 벨로그는 어떤 식으로 진행하는게 좋을까?
		- 이력서에 링크를 남기는게 좋을까?

# 생각등대컴퍼니

- 이유 없이 작성하지 말자


- 문제, 해결, 결과에 맞게 작성하자
	- 각 3줄씩 작성해보자
- 구현하기는 이제는 기본, 생각의 흐름이 중요하다
	- 생각의 흐름을 시각화하자
		- 아키텍처
		- 데이터플로우
		- 시퀀스 다이어그램


- 프로젝트는 2개만 하자?
	- 프로젝트 안에 질문이 많다면 충분하다
- 문제 해결은 문제 예방과 같다
	- 문제를 예방하기 위한 방향성을 어필
- 면접관의 의도를 파악하자?


- 개인 프로젝트에서는 전체적인 플로우를 경험하는 것이 중요하다
	- 백엔드 & 프론트엔드 & 데브옵스
- 기본기 70 : 기술 30
	- 기본기 중에서도 30은 deep한 기본기를 작성
	- 예시 (일단 예시일뿐...)
		- 기본기 : 비동기, 레디스, 로컬 캐시
		- 기술 : mongoDB, kafka
		- 딥한 기본기 : JVM 튜닝
- 포트폴리오는 가독성이 생명
	- 열정, 열심, 꽃... 이런 추상적이고 쓸데없는 내용은 삭제
- 포트폴리오는 각 프로젝트에 7줄 정도 작성
	- 각 한 줄에 문제 해결 결과가 나오게 작성
- 이모티콘 다 날리기


- 직접 홈페이지에 들어가서 회사, 팀에 맞는 이력서를 수정한다

- 문제가 없었을 경우에는 트래픽을 부하시켜 간접 테스트?

- 면접 마지막 말은 무엇을 해야될까?
	- "없습니다"가 최악이다.
	- 제가 팀원에 들어가게 된다면 무슨 역할을 하나요?


- 급하게 하지말자, 면접관의 심리를 파악
	- 이력서 2~3장, 자기소개 2~3줄, 문제 해결 단 1줄 


- 이력서를 다 만들 때 똑같다... 나만의 무기 만들기?
	- 서비스 기업의 테크 블로그는 2~3주동안 작성함
	- 1달 동안 하나의 포스팅을 온 힘을 다해서 만들어보자
		- 흘리듯 툭 링크 던지기


- AI를 한스푼 더하자
	- 현재 실무에서 AI를 활용해서 어떻게 생산성을 높이는지에 대해 관심이 높다.
	- 하나를 AI를 이용한 프로젝트를 만들어 보자.
	- LLM 사용에 대해 얘기할 수 있는 것도 좋다.


- 이력서에도 지원동기를 작성하자
