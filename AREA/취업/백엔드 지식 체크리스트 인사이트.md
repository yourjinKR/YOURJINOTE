# 최소 요구 지식 리스트 1

해당 체크 리스트는 신입 백엔드 개발자로써 기초 of 기초 지식을 나열함 (chatGPT)

## 1️⃣ Java (가장 중요 ⭐⭐⭐⭐⭐)

### ✅ 문법 & 기본기

-  클래스 / 객체 / 인스턴스 차이를 설명할 수 있다
-  접근 제어자(public / protected / default / private)를 목적에 맞게 사용한다
-  `final`의 의미를 설명할 수 있다
-  static의 의미와 사용 시점을 설명할 수 있다

### ✅ 객체지향

-  객체지향 4대 원칙을 **예시 코드와 함께** 설명할 수 있다
-  인터페이스와 추상 클래스의 차이를 설명할 수 있다
-  다형성이 왜 필요한지 설명할 수 있다
-  “역할과 구현 분리”가 무엇인지 이해하고 있다

### ✅ 컬렉션 & 제네릭

-  List / Set / Map의 차이를 알고 있다
-  ArrayList vs LinkedList 차이를 설명할 수 있다
-  HashMap의 동작 원리를 대략적으로 설명할 수 있다
-  제네릭을 왜 쓰는지 설명할 수 있다

### ✅ 예외

-  Checked / Unchecked Exception 차이를 설명할 수 있다
-  예외를 언제 던지고 언제 잡아야 하는지 안다
-  실무에서 RuntimeException을 주로 쓰는 이유를 설명할 수 있다

### ✅ JVM

-  JVM 메모리 구조(Heap / Stack / Metaspace)를 설명할 수 있다
-  GC가 왜 필요한지 설명할 수 있다
-  참조 타입과 기본 타입 차이를 안다

👉 **이 중 80% 이상 설명 가능하면 “Java 합격선”**
<br>
## 2️⃣ Spring (핵심 ⭐⭐⭐⭐⭐)

### ✅ Spring Core

-  Spring이 왜 등장했는지 설명할 수 있다
-  IoC / DI 개념을 말로 설명할 수 있다
-  Bean이 무엇인지 알고 있다
-  `@Component`, `@Service`, `@Repository` 차이를 안다

### ✅ Spring MVC

-  요청 → 컨트롤러 → 서비스 → 레포지토리 흐름을 설명할 수 있다
-  `@Controller` vs `@RestController` 차이를 안다
-  `@RequestMapping`, `@GetMapping` 등을 쓸 수 있다
-  DTO를 왜 사용하는지 설명할 수 있다

### ✅ 트랜잭션

-  트랜잭션이 무엇인지 설명할 수 있다
-  ACID를 말로 설명할 수 있다
-  `@Transactional`의 역할을 안다
-  트랜잭션이 서비스 계층에 있어야 하는 이유를 안다

👉 **CRUD 프로젝트 1개 이상이면 합격선**
<br>
## 3️⃣ DB / SQL (⭐⭐⭐⭐⭐)

### ✅ DB 기본 개념

-  PK / FK 차이를 설명할 수 있다
-  정규화가 왜 필요한지 설명할 수 있다
-  ERD를 보고 테이블 구조를 이해할 수 있다

### ✅ SQL

-  SELECT / INSERT / UPDATE / DELETE 작성 가능
-  JOIN을 사용할 수 있다
-  GROUP BY / HAVING을 사용할 수 있다
-  서브쿼리를 써본 적이 있다

### ✅ 트랜잭션 & 성능

-  트랜잭션이 DB에 왜 필요한지 설명할 수 있다
-  인덱스가 무엇인지 알고 있다
-  인덱스를 언제 써야 하는지 안다

👉 **SQL로 CRUD + JOIN 가능하면 합격선**
<br>
## 4️⃣ JPA / MyBatis (⭐⭐⭐⭐)

### ✅ ORM 이해

-  JPA가 무엇인지 설명할 수 있다
-  ORM의 장단점을 말할 수 있다
-  Entity가 무엇인지 알고 있다

### ✅ JPA 기본

-  `@Entity`, `@Id`, `@GeneratedValue` 사용 가능
-  연관관계(OneToMany, ManyToOne) 이해
-  Fetch 전략(LAZY / EAGER) 차이를 안다
-  N+1 문제를 들어본 적 있고 설명 가능

👉 **JPA로 CRUD 가능하면 충분**
<br>
## 5️⃣ Linux / 서버 기초 (⭐⭐⭐)

### ✅ Linux

-  기본 명령어 사용 가능 (`ls`, `cd`, `ps`, `top`)
-  로그 확인 방법을 안다
-  포트 확인 가능
-  실행 중인 서버 종료 가능
    
### ✅ 서버 이해

-  WAS와 Web Server 차이를 안다
-  8080 포트가 뭔지 안다
-  Spring Boot jar 실행해본 적 있다

👉 **“서버 무섭지 않다” 수준이면 OK**
<br>
## 6️⃣ REST API & 설계 감각 (⭐⭐⭐⭐)

-  REST API가 무엇인지 설명할 수 있다
-  HTTP Method 의미를 안다 (GET / POST / PUT / DELETE)
-  상태 코드(200 / 400 / 404 / 500)를 안다
-  URI 설계 경험이 있다
<br>
## 7️⃣ 프로젝트 (가장 중요 ⭐⭐⭐⭐⭐)

### ✅ 필수 조건

-  개인 프로젝트 1개 이상
-  CRUD 포함
-  DB 연동
-  REST API 형태
<br>
### ✅ 있으면 강력

-  로그인 / 회원
-  트랜잭션 처리
-  예외 처리 구조
-  README 정리

👉 **프로젝트 설명이 막힘없이 가능하면 합격선**

# 최소 요구 지식 리스트 3

### 1. Java 기본기 ☕

Java는 백엔드 개발의 근간이 되는 언어이니만큼 탄탄한 기본기가 중요해요.

- **[ ] 객체 지향 프로그래밍 (OOP) 개념 이해:**
    - 추상화, 캡슐화, 상속, 다형성 개념을 명확히 이해하고 실제 코드에 적용할 수 있어야 해요.
    - _예시:_ 인터페이스와 추상 클래스의 차이, 오버라이딩과 오버로딩의 차이 등을 설명할 수 있어야 합니다.
- **[ ] Java 자료구조:**
    - `List`, `Set`, `Map` 등 기본적인 컬렉션 프레임워크 사용법과 각 자료구조의 특징 (성능, 사용 사례)을 이해해야 해요.
    - _예시:_ `ArrayList`와 `LinkedList`의 차이점, `HashMap`의 내부 동작 방식 (간단하게라도)을 이해하면 좋아요.
- **[ ] Java 동시성/병행성 (Concurrency):**
    - `Thread` 기본 개념, `synchronized` 키워드, `volatile` 키워드 등 기본적인 동시성 제어 방법을 알아두면 좋아요. (신입에게는 깊은 이해보다 개념 이해 정도를 요구하는 경우가 많아요.)
- **[ ] 예외 처리 (Exception Handling):**
    - `try-catch-finally` 블록 사용법, `checked` / `unchecked` 예외의 차이, 커스텀 예외 생성 및 처리 방법을 알아야 해요.
- **[ ] Java 8+ Stream API:**
    - 간결하고 함수형 프로그래밍 스타일로 데이터를 처리하는 Stream API 활용 능력이 있으면 코드가 훨씬 깔끔해져요.
    - _예시:_ `map`, `filter`, `reduce` 등의 기본 메소드 사용법을 익혀두세요.

### 2. Spring Framework & Spring Boot 🍃

Spring은 Java 백엔드 개발의 표준이라고 할 수 있죠! Spring Boot를 통해 개발 생산성을 높이는 것이 중요해요.

- **[ ] Spring IoC (Inversion of Control) & DI (Dependency Injection):**
    - 스프링의 핵심 원리인 IoC 컨테이너와 DI 개념을 확실히 이해해야 해요. Bean의 생명주기, `@Component`, `@Autowired`, `@Bean` 등의 어노테이션 사용법을 익혀두세요.
    - _예시:_ 의존성 주입이 왜 필요한지, 스프링이 어떻게 객체를 관리하는지 설명할 수 있어야 합니다.
- **[ ] Spring MVC:**
    - 웹 요청 처리 흐름 (Controller, Service, Repository 계층)을 이해하고, `@Controller`, `@RequestMapping`, `@RequestParam`, `@PathVariable` 등 관련 어노테이션을 활용할 수 있어야 해요.
- **[ ] Spring Boot:**
    - Spring Boot의 장점 (자동 설정, 내장 서버 등)을 이해하고, `application.properties` 또는 `application.yml` 설정 파일 관리 방법을 알아야 해요.
    - _예시:_ Spring Boot Starter 의존성 관리, 외부 설정 적용 방법 등을 익혀두세요.
- **[ ] Spring Data JPA:**
    - ORM (Object-Relational Mapping) 개념을 이해하고, Spring Data JPA를 이용해 데이터베이스 연동을 할 수 있어야 해요. `@Entity`, `@Repository`, JpaRepository 인터페이스 활용법을 익혀두세요.
    - _예시:_ 기본적인 CRUD 작업, 간단한 쿼리 메소드 작성, `@Query` 어노테이션 사용법 등을 알아두면 좋아요.
- **[ ] Spring Security (기본):**
    - 인증(Authentication)과 인가(Authorization)의 개념을 이해하고, Spring Security를 이용해 기본적인 로그인/로그아웃 기능과 권한 관리를 구현할 수 있어야 해요. (신입에게는 복잡한 설정보다 기본 개념과 적용 방법을 요구하는 경우가 많습니다.)

### 3. 데이터베이스 (RDBMS) 🗄️

데이터를 다루는 백엔드 개발자에게 데이터베이스 지식은 필수죠!

- **[ ] SQL 기본 문법:**
    - `SELECT`, `INSERT`, `UPDATE`, `DELETE` 등 기본적인 DML 문법은 당연히 알아야 하고, `JOIN` (Inner, Left, Right), `GROUP BY`, `HAVING`, 서브쿼리 등도 능숙하게 사용할 수 있어야 해요.
- **[ ] 데이터베이스 기본 개념:**
    - 정규화, 트랜잭션 (ACID), 인덱스, 제약 조건 (Primary Key, Foreign Key, Unique) 등의 개념을 이해하고 있어야 해요.
    - _예시:_ 인덱스가 왜 성능 향상에 도움이 되는지, 트랜잭션이 왜 중요한지 설명할 수 있으면 좋습니다.
- **[ ] 관계형 데이터베이스 모델링:**
    - 요구사항에 맞춰 테이블을 설계하고 테이블 간의 관계 (1:1, 1:N, N:M)를 정의할 수 있어야 해요.

### 4. API & 통신 🌐

클라이언트와 서버가 소통하는 방식을 이해하는 것이 중요해요.

- **[ ] RESTful API 개념:**
    - REST의 기본 원칙 (Stateless, Cacheable, Uniform Interface 등)을 이해하고, HTTP 메소드 (GET, POST, PUT, DELETE)를 올바르게 사용하는 방법을 알아야 해요.
    - _예시:_ 각 HTTP 메소드가 어떤 의미를 가지는지, 리소스 기반으로 API를 설계하는 방법을 이해해야 합니다.
- **[ ] HTTP & HTTPS:**
    - HTTP의 요청/응답 구조, 상태 코드 (2xx, 4xx, 5xx 등)의 의미를 이해하고, HTTPS의 기본 개념을 알아두면 좋아요.
- **[ ] JSON 데이터 형식:**
    - API 통신에서 가장 널리 사용되는 JSON 형식의 구조를 이해하고, Java 객체와 JSON 간의 직렬화/역직렬화 (Serialization/Deserialization) 방법을 알아야 해요. (Jackson 라이브러리 등)

### 5. 기타 & 추가 역량 💡

이 외에도 알아두면 좋은 기술들이 있어요.

- **[ ] Git & GitHub:**
    - 버전 관리 시스템인 Git의 기본적인 사용법 (commit, push, pull, branch, merge)은 필수입니다. 협업을 위한 GitHub 사용법도 익혀두세요.
- **[ ] 빌드 도구 (Maven/Gradle):**
    - 프로젝트 의존성 관리 및 빌드 자동화를 위한 Maven 또는 Gradle 사용법을 익혀두세요. (Spring Boot는 보통 Gradle을 많이 사용해요.)
- **[ ] 단위 테스트 (JUnit):**
    - 작성한 코드가 의도대로 동작하는지 검증하는 단위 테스트 작성 능력이 중요해요. JUnit을 이용한 테스트 코드 작성법을 익혀두세요.
- **[ ] 간단한 웹 서버/WAS 이해:**
    - Tomcat과 같은 WAS의 역할과 동작 방식에 대한 기본적인 이해가 있으면 좋습니다. (Spring Boot는 내장 서버를 사용하지만, 기본적인 개념은 알아두면 좋아요.)
