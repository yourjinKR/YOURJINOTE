# JVM의 메모리 구조를 설명해주세요

[JVM의 메모리 구조](../../../../RESOURCE/JAVA/Java-Runtime-Data-Areas.md)는 크게 두 가지로 나뉩니다. 모든 스레드가 공유하며 객체와 클래스 메타데이터를 저장하는 힙(Heap)과 메서드 영역(Method Area)이 있고, 각 스레드가 생성될 때마다 개별적으로 할당되는 pc 레지스터, JVM 스택(Stack), 네이티브 메서드 스택이 있습니다. 스택 내부에는 메서드 호출마다 프레임(Frame)이 생성되어 지역 변수와 연산 결과를 저장합니다.

- 스레드별 데이터 영역
	- **pc 레지스터** : 현재 스레드가 실행중인 JVM 명령어의 주소
	- **JVM 스택** : 메서드 호출과 반환, 지역 변수, 부분 결과 등을 저장, `StackOverflowError`
	- **네이티브 메서드 스택** : 네이티브 메서드를 실행하기 위해 사용되는 영역
- 모든 스레드가 공유하는 데이터 영역
	- **힙** : 객체와 배열이 할당되는 영역, 가비지 컬렉터가 메모리 정리, `OutOfMemoryError`
	- **메서드 영역** : 클래스 단위의 정보 저장, 논리적으로는 힙의 일부분
	- **런타임 상수 풀** : 메서드 영역의 일부로 할당, 클래스 파일 내의 `constant_pool` 테이블을 런타임에 표현 (클래스 버전, 필드, 메서드, 인터페이스 등)
