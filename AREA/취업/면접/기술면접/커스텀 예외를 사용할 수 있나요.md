# 커스텀 예외를 사용할 수 있나요

Java에서 커스텀 예외는 다음과 같이 만들 수 있습니다.

```java
public class CustomException extends RuntimeException { }
```

커스텀 예외를 만들 때 가장 중요한 결정은 `Exception`을 상속받을 것인가, `RuntimeException`을 상속받을 것인가입니다.

JLS의 일반적인 권장사항은 Checked Exception을 사용하는 것이지만, 현대 백엔드 개발(특히 Spring Framework)에서는 커스텀 예외를 주로 RuntimeException을 상속받는 Unchecked Exception으로 만듭니다. Checked Exception을 남용하면 모든 상위 메서드 시그니처에 throws를 명시해야 하므로 코드가 지저분해지고(OCP 위반), 트랜잭션 롤백 처리 등에서 불편함이 발생하기 때문입니다.

### 비지니스 로직에서 RuntimeException를 선호하는 이유

가장 큰 이유는 **불필요한 예외 처리 코드의 전파(throws)를 막고 코드의 가독성을 높이기 위해**
