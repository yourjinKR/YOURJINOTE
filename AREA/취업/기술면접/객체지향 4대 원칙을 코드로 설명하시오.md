# 객체지향 4대 원칙을 코드로 설명하시오

객체지향의 4대 원칙은 캡슐화, 상속, 다형성, 추상화입니다.

[캡슐화](../../../RESOURCE/CS/Encapsulation-캡슐화.md)는 연관된 속성과 기능을 하나의 캡슐로 만드는 것입니다.  
또한 데이터를 외부로부터 보호하여 **객체 고유의 독립성**과 **책임 영역**을 지킵니다.

[상속](../../../Inheritance-상속.md)은 기존 클래스의 **속성과 기능을 이어받아 사용**하는 것입니다.
**자바에서는 `extends` 키워드**를 통해 해당 클래스의 멤버 변수와 메서드를 사용할 수 있습니다.

[다형성](../../../RESOURCE/CS/Polymorphism-다형성.md)은 **하나의 데이터가 맥락에 따라 여러 기능을 수행**할 수 있음을 말합니다.  
대표적으로 자바에서는 **메서드 오버라이딩**을 통해 구현할 수 있습니다.

[추상화](../../../RESOURCE/CS/Abstraction-추상화.md)는 **중요한 정보만 표현**한 것을 말합니다.
자바에서는 **인터페이스를 선언하고 구현체를 만드는 과정**을 통해 구현할 수 있습니다.

## 자바 코드로 설명

- 캡슐화 : getter/setter
- 상속 : extends
- 다형성 : override
- 추상화 : interface implements
