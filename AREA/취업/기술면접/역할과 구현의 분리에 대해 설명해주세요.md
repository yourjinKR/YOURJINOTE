역할은 기능의 명세이고, 구현은 기능의 실제 처리 방식입니다.  
이 두개의 개념을 분리한다는 것은 객체 지향 설계 원칙 중 하나입니다.  
자바에서 역할은 [인터페이스](../../../RESOURCE/JAVA/Java-인터페이스-Interface.md)이며 구현은 인터페이스를 구현한 클래스입니다.

역할과 구현의 분리를 통해 다음과 같은 이점을 얻습니다.

- **클라이언트**는 인터페이스(역할)만 알면 된다.
- **클라이언트**는 구현 대상의 **내부 구조를 몰라**도 된다.
- **클라이언트**는 구현 대상의 **내부 구조가 변경**되어도 영향을 받지 않음
- **클라이언트**는 구현 **대상 자체를 변경**해도 영향을 받지 않음

public interface SocialUserInfoExtractor {  
    SocialProviderType supports();  
    SocialUserInfo extract(OAuth2User oAuth2User);  
}
