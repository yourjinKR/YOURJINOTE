### 최초 프롬프트 

gpt가 추천한 최초 프롬프트는 다음과 같았다.

```
너는 시니어 프론트엔드 개발자다.  
나는 신입/주니어 백엔드 개발자 취준생이며, 채용 담당자가 30초 안에 강점을 파악할 수 있는 단일 랜딩 포트폴리오 웹사이트를 만들고 싶다.
목표: Next.js(App Router) + TypeScript + Tailwind CSS로 “단일 랜딩 포트폴리오”를 만든다.
서버 없이 배포(Vercel)까지 고려한다.

요구사항:
1) Next.js App Router 사용: src/app/layout.tsx, src/app/page.tsx
2) 스타일은 Tailwind CSS만 사용한다 (styled-components 사용 금지)
   - tailwind.config.ts의 theme.extend에 색상/폰트/반응형 토큰 정의
   - globals.css는 Tailwind base/components/utilities 유지 + 최소한의 기본 스타일만 추가
3) 콘텐츠는 src/content/profile.ts 한 파일에서만 관리한다.
   - 모든 섹션 텍스트/리스트/프로젝트/스킬/링크를 profile.ts에서 제공
   - UI 컴포넌트는 profile.ts 데이터를 읽어 렌더링만 한다.
4) 단일 페이지 스크롤 섹션 순서:
   - 성장과정 → 자기소개 → 간단 프로필 → EXPERIENCE → Skills → 개발자에 대한 생각 → Projects
5) 공통 UI:
   - 상단 고정 네비게이션(섹션 앵커 이동 + 현재 섹션 active 표시)
   - 섹션 컴포넌트 분리: src/components/sections/*
   - Projects: 카드 목록 + 클릭 시 모달로 상세 표시
     * 접근성: aria, ESC 닫기, 바깥 클릭 닫기, 모달 열릴 때 포커스 처리
6) UX:
   - 채용 포트폴리오 느낌(미니멀, 과한 효과 X)
   - 모바일 우선 반응형
   - IntersectionObserver 기반 scroll reveal(fade/slide) 훅 제공
7) 산출물:
   - 파일 트리 먼저 제시
   - 필요한 파일 전체 코드(각 파일 경로를 제목으로, 코드블록으로)
   - 실행 방법(npm run dev/build/start)과 Vercel 배포 방법 요약

제약:
- 외부 UI 프레임워크(MUI, Chakra 등) 사용 금지
- Tailwind만 사용
- 콘텐츠 수정은 profile.ts만 바꾸면 되도록 설계
```

### 같은 문제가 지속적으로 발생

마크다운 형식으로 지침서를 만들어 달라는 요구를 했다.

```
› 현재 문제점이 많네 프로젝트 전반적인 지침서를 만드는게 좋을거 같아.
```

그러나 글자가 깨진 형태로 답변하거나 이를 수정하면 다른 지침서의 문자가 다시 깨지는 문제가 발생했다.

```

  서를 만들어야 할까? 계속 지속적으로 똑같은 문제가 발생하기에.
```

```
› 지금 AGENT.md는 정상적으로 수정됐으나 나머지 지침서들이 멀쩡했는데 다시 깨지는 중
  임. 원인 분석하고 이러한 문제가 다시 발생하지 않도록 너가 방안을 마련해.
```

이후로도 디자인 수정 요청을 지속했으나 요구사항을 적용시켜주지 않았다.


## 2차 시도

1차 시도에서 어느 순간 같은 문제가 반복되거나 `디자인 전반적으로 깔끔하게 수정 바람` 와 같은 요구가 반복적으로 적용이 되지 않았고 작업의 정체를 느꼈다.

### 설계부터 수행

디자인을 수정하는 과정에서 명령하는대로 잘 되지 않아서 새로 생성하고 다시 시도하도록 했다.
처음엔 1차 초기 프롬프트를 굉장히 많이 전달했었다.  
이 보다는 계획 단계부터 에이전트와 같이 수행하는 것을 시도해봤다. (프로젝트의 규모가 작기 때문에)  

> 프로젝트 빌드 후 목표를 간단히 언급했다.  
> 그리고 기초적인 문서와 타입을 관리하는 파일만을 생성했다.

```
신입 개발자 취준생 포트폴리오를 만들거야. 
우선, AGENTS.md, content/profile.ts, TODO.md를 생성해. 먼저 초안만 만들고 파일 내용 보여줘.
```

UI를 우선적으로 개선하기 보다는 어떤 데이터를 보여줄 것이냐를 확실하게 정하고 개발하는 것이 좋다고 생각하여 아래와 같이 제안했고 기존에 노션으로 작성한 포트폴리오 전체 분량을 그대로 전달했다.

```
일단 내가 필요한 기본 정보들을 정리해보자. 내가 노션에 정리한 포트폴리오가 있는데 이를 그대로 보내주면 될까? 아니면 .md에다가 저장하는게 좋을까?
```

> 노션에 작성했던 포트폴리오를 그대로 복붙하여 보내줬고 이를 `raw-profile.md`로 관리함.

이를 기반으로 `profile.ts`에 내가 원하는 양식의 프로필 타입이 정의됐다.

### 작업 순서 관리

하나의 작업을 수행하고 `TODO.md`를 최신화 하면서 작업 순서를 유지하도록 했다.

```
지금까지 진행한 업무 TODO 최신화 바람.
```

```
다음에 진행하면 좋을 업무들 알려줘
```

이후로는 에이전트가 업무 순서에 맞게 작업을 수행했으며 중간에 작업이 추가되더라도 `TODO.md`에 최신화하여 작업 과정을 수정하고 실행하는 것을 확인할 수 있었다.

또한 진행 과정 중 일부 TODO를 볼때 이 업무가 무엇인지 파악이 안되는 경우가 있어 간단하게 설명을  듣고 진행했다.

![Pasted image 20260220160413](GALLERY/Pasted%20image%2020260220160413.png)

### 마무리

```
일단 여기서 1차적으로 마무리 짓자. (vercel로 우선 1차적으로 배포할 예정임) 
다음 업무는 디자인(CSS 스타일) 위주로 리팩토링할거 같아. 
지금까지한 업무들을 고려하여 AGENT.md 와 TODO.md를 최신화하고 추가적으로 문서화할게 있다면 추가로 작업해줘. 
컨텍스트가 초기화되더라도 작업 맥락을 잃지 않도록.
```

디자인(CSS 스타일) 컨셉과 세부 디자인 작업을 분리하고 싶어 다음과 같이 요구했고 현재 컨텍스트에서 작업한 내용들을 관리하도록 했다.

# 느낀 점

### 작업 관리

에이전트에게 작업 계획을 맡기고 이에 대해 순차적으로 수행하면서 최신화 하는 과정의 중요성을 느꼈다.  
더 복잡한 문제를 해결하기 위해서는 AI에게 작업 계획과 이를 수행하는 프로세스를 체계화 할 필요성을 느꼈다. 

### 모르는 분야에 대한 방황

확실히 한번도 사용해보지 않았거나 익숙하지 않은 기술들(next.js, tailwind, typescript)이라 스스로 문제를 파악하기 어려웠으며 자연스럽게 AI에게 의존하게 됐다.  
당연하게도 수정 작업 완료 후 작업에 대한 설명을 읽어도 문제의 원인이 무엇이며 어떻게 해결했는지를 이해하지 못했다.  
반면 조금이라도 사전에 경험이 있다면 충분히 해결할 수 있음을 느꼈다.  

# 궁금한 점

- 지속적으로 요구사항이 적용되지 않을 때는 어떻게 대처하시는지
- AI에게 코딩을 맡기고 병렬적으로 어떤 업무를 수행하시는지
