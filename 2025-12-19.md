# 스택과 큐

### ADT (Abstract Data Type)

- 추상자료형
- 개념적으로 어떤 동작이 있는지만 정의
- 구현에 대해서는 다루지 않음

### Data Structure

- 자료구조
- ADT에서 정의된 동작을 실제로 구현한 것


## Stack

- LIFO
- 함수 호출 스택


## Queue

- FIFO
- 큐 주요 동작
	- enqueue
	- dequeue
	- peek
- producer/consumer architecture
- 큐가 항상 FIFO를 의미하지 않는다.
	- priority queue : 우선순위 기반으로 먼저 나옴
	- "대기공간"의 개념으로 사용되기도 함

## 대표 문제 상황

### StackOverFlowError와 재귀 함수
- 스택 메모리를 다 사용할때 사용

### OutOfMemoryError

- Java의 힙 메모리를 다 썼을 때 발생
	- 큐에 데이터가 계속 쌓이기만 할때 발생
		- 큐 사이즈를 고정
			- 큐가 다 찼을때 어떻게 할 것인가?
				- exception 던지기
				- 특별한 값을 반환
				- 성골할 때까지 영원히 스레드 블락
				- 제한된 시간만 블락되고 그래도 안되면 포기
