# Jvm의 메모리 구조를 설명해주세요

JVM의 메모리 구조는 모든 스레드가 공유하는 힙과 메서드 영역이 있으며,
스레드별로 각각 관리되는 pc레지스터, JVM 스택, 네이티브 메서드 스택이 있습니다.

## 스레드별 데이터 영역 (pc 레지스터, 스택, 네이티브 메서드 스택)
pc 레지스터는 현재 스레드가 실행중인 JVM 명령어의 주소를 저장합니다.
스택은 메서드 호출과 반환, 지역 변수 등을 저장합니다.
네이티브 메서드 스택은 네이티브 메서드를 실행하기 위해 사용되는 영역입니다.

## 모든 스레드가 공유하는  데이터 영역
Heap은 객체와 배열을 저장하며 GC가 메모리 관리를 수행합니다.
Method Area는 타입 정보, 상수, 정적 변수 정보가 저장되는 영역입니다. Heap의 일부분입니다.
런타임 상수 풀은 Method Area의 일부로 할당되며 클래스 파일 내의 constant_pool 테이블을 런타임에 표현합니다.


# GC가 필요한 이유에 대해 설명해주세요

C/C++과 같은 프로그래밍 언어는 개발자가 언어 레벨에서 메모리를 직접 관리했습니다.
이 과정에서 개발자의 실수는 메모리 누수 문제를 야기했습니다.
자바에서는 언어 레벨에서 메모리에 대한 직접적인 접근을 막고 메모리 관리를 GC가 관리하게 하여
개발자가 개발에만 집중할 수 있게 하도록 설계되었습니다.


# 기본 타입과  참조 타입의 차이를 설명해주세요

기본 타입은 int, float, boolean과 같이 자바 언어에서 미리 정의된 타입들입니다.
기본 타입의 변수들은 메모리에 원시 값을 직접 저장합니다.

참조 타입은 객체와 배열과 같이 기본 타입을 제외한 나머지들이며,
메모리에 값을 직접 저장하는 것이 아닌 Heap 메모리에 동적으로 생성된 객체를 가리키는 포인터를 저장합니다.
